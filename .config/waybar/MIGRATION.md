### Система цветов (STEP 2)
- **`themes/palette.css`** — единый источник цветов. Все модули берут цвет отсюда
- **`style.css`** — использует только `@wb-*` переменные, никаких хардкоженных цветов (кроме семантических: погода, voxtype)
- Все 5 preset-тем конвертированы в формат `@define-color`
- `change-theme.sh` / `cycle-theme.sh` обновлены: копируют preset → `palette.css`

### Структура цветов в palette.css
```
Base palette       → color-bg, color-surface, color-overlay, color-muted, color-text
Accent slots (×5)  → color-accent-1 .. color-accent-5
Semantic aliases   → wb-bar-bg, wb-module-bg, wb-module-fg,
                     wb-active-bg,   wb-active-fg,
                     wb-inactive-bg, wb-inactive-fg,
                     wb-hover-bg,    wb-hover-fg,
                     wb-border, wb-muted
```

## Следующие шаги: интеграция с обоями

### Что должен делать скрипт `scripts/wallpaper-colors.sh`

1. Получить путь к текущему обою
2. Извлечь акцентные цвета (5 штук) + базовые (bg, text)
3. Записать новый `themes/palette.css`
4. Перезапустить waybar: `pkill waybar && waybar &`

### Ожидаемые входные данные

Скрипту нужно на выход получить **минимум 7 цветов** в формате hex:

| Переменная       | Роль                            | Пример     |
|------------------|---------------------------------|------------|
| `color-bg`       | Тёмная база (bar bg, active fg) | `#1b1e25`  |
| `color-text`     | Основной текст модулей          | `#ebddbc`  |
| `color-accent-1` | Главный акцент (часы, voxtype)  | `#DEC386`  |
| `color-accent-2` | Вторичный (сеть, bt, audio)     | `#EAB2C2`  |
| `color-accent-3` | Третичный (backlight, battery)  | `#83A6BF`  |
| `color-accent-4` | Hover цвет                      | `#B28999`  |
| `color-accent-5` | Workspace hover                 | `#D5A5AA`  |

`color-surface`, `color-overlay`, `color-muted` можно вычислить как осветлённые/затемнённые варианты `color-bg`.

### Формат вывода — шаблон palette.css

Скрипт должен сгенерировать файл следующего вида:

```css
/* Generated by wallpaper-colors.sh from: /path/to/wallpaper.jpg */
@define-color color-bg       #1b1e25;
@define-color color-surface  #242830;
@define-color color-overlay  #2b2f38;
@define-color color-muted    #6e6a86;
@define-color color-text     #ebddbc;

@define-color color-accent-1 #DEC386;
@define-color color-accent-2 #EAB2C2;
@define-color color-accent-3 #83A6BF;
@define-color color-accent-4 #B28999;
@define-color color-accent-5 #D5A5AA;

@define-color wb-bar-bg     rgba(27, 30, 37, 1);
@define-color wb-module-bg  #242830;
@define-color wb-module-fg  #ebddbc;
@define-color wb-active-bg  #DEC386;
@define-color wb-active-fg  #1b1e25;
@define-color wb-hover-bg   #B28999;
@define-color wb-hover-fg   #1b1e25;
@define-color wb-border     #2b2f38;
@define-color wb-muted      #6e6a86;
```

### Варианты бэкенда для извлечения цветов

| Инструмент   | Язык   | Что даёт                          | Установка               |
|--------------|--------|-----------------------------------|-------------------------|
| **matugen**  | Rust   | Material You — 5 тональных ролей  | `cargo install matugen` |
| **pywal**    | Python | 16-цветная палитра из доминант    | `pip install pywal`     |
| **colorthief** | Python | N доминантных цветов             | `pip install colorthief`|

**Рекомендация**: matugen — он сразу выдаёт роли (`primary`, `secondary`, `tertiary`, `on_primary` и т.д.) которые ложатся 1:1 на слоты `color-accent-*`.

### Пример маппинга matugen → palette

```
matugen primary         → color-accent-1  (главный акцент)
matugen secondary       → color-accent-2  (вторичный)
matugen tertiary        → color-accent-3  (третичный)
matugen secondary_fixed → color-accent-4  (hover)
matugen surface_variant → color-accent-5  (workspace hover)
matugen background      → color-bg
matugen on_background   → color-text
matugen surface         → color-surface
```

### Пример вызова

```bash
# Вручную для конкретного обоя
~/.config/waybar/scripts/wallpaper-colors.sh ~/Pictures/wallpaper.jpg

# Автоматически после смены обоев (добавить в hyprpaper/swww хук)
exec-once = swww-daemon
# после смены: swww img wallpaper.jpg && wallpaper-colors.sh wallpaper.jpg
```

### Точка интеграции с Hyprland

Скрипт нужно вызывать из того же места, где меняется обой. Типичные варианты:

- **swww**: обернуть `swww img` в shell-функцию/скрипт
- **hyprpaper**: добавить `exec` после смены через `hyprctl hyprpaper wallpaper`
- **waypaper**: хук `--fill` / post-command
